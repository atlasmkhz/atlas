# ATLAS UI 추가 작업 — 구현 완료 보고

## v1.2 변경사항 (이번 수정 — 응원의 기록 전체 제거)

사용자 판단에 따라 「응원의 기록」 기능을 전체 제거했다. 다음이 삭제되었다:

- `index.html`: 응원의 기록 팝업 마크업(`encouragementModal` 등), 하단 푸터의 응원의 기록 버튼
- `js/ui.js`: 응원의 기록 열기/닫기 로직 전체
- `js/guestbook.js`: 파일 자체 삭제
- `worker/`: Cloudflare Worker 디렉토리 전체 삭제
- `admin-records/`: 관리자 승인 페이지 디렉토리 전체 삭제
- `css/style.css`: `.encouragement-*` 전체 규칙, `.bottom-fixed-sep` 삭제

결과적으로 **Cloudflare Worker/KV 도입 자체가 더 이상 필요 없다** — 이제 정적 Pages
배포만으로 전체 기능(소개 팝업 + 하단 고정 이메일 안내)이 완성된다.

남은 것: 소개 팝업(닫기/오늘 하루 열지 않기) + 하단 고정 푸터(이메일 버튼 1개) +
이메일 표시 팝업. 전부 정적 HTML/CSS/JS로만 동작하며 서버/API가 필요 없다.

## v1.1 변경사항 (이전 수정)

이전 버전(v1.0)은 이메일/응원의 기록이 **소개 팝업 내부**에 있었으나,
"팝업과 무관하게 항상 보이는 하단 고정 영역"으로 구조를 다시 짰다.

| 항목 | v1.0 | v1.1 |
|---|---|---|
| 이메일 위치 | 소개 팝업 내부 하단 | 지도 화면 자체의 `#bottomFixedBar` (팝업과 독립) |
| 소개 팝업 닫기 방식 | X 버튼 하나 | 「오늘 하루 열지 않기」(자정 만료, localStorage) + 「닫기」(세션 유지, sessionStorage) 2버튼 |
| 이메일 노출 방식 | mailto 링크 텍스트 | 클릭 시 별도 "이메일 표시 팝업"(`#emailPopup`)이 뜸 |
| 지도/타임라인 레이아웃 | 변경 없음 | 하단 고정 푸터 높이(`--bottom-bar-height:42px`)만큼 `#map`·`.timeline`을 위로 띄움 |
| 푸터 z-index | (해당없음) | `1250` — `.info-panel`(1200)보다 위로, 카드가 열려 있어도 항상 클릭 가능 |

### 발견하고 고친 문제
1. **타임라인 힌트 텍스트 가려짐**: 푸터를 그냥 얹었더니 "슬라이더로 연도 이동…" 안내문이
   가려짐 → `#map`과 `.timeline`의 `bottom` 좌표를 푸터 높이만큼 띄워서 해결.
2. **카드(정보 패널) 열렸을 때 푸터 버튼 클릭 불가**: 데스크톱에서 우측 420px 폭의
   정보 패널(`z-index:1200`)이 푸터(`z-index:900`)를 덮어버려, 카드가 열린 상태에서는
   "응원의 기록" 버튼을 누를 수 없었음(검증 항목 ④ 위반) → 푸터 z-index를 1250으로
   올려 항상 위에 뜨게 수정. Playwright로 카드 열린 상태에서 실제 클릭까지 재현해 확인함.

## 현재 구현 범위 (v1.2 최종)

순수 정적 사이트(Cloudflare Pages)만으로 동작한다. 서버/Worker/KV/DB 전부 없음.

## 새로 추가/수정된 파일

| 파일 | 내용 |
|---|---|
| `index.html` | 소개 팝업(`welcomeModal`, 2버튼) + 하단 고정 푸터(`bottomFixedBar`, 이메일 버튼만) + 이메일 팝업(`emailPopup`) 마크업 |
| `css/style.css` | 위 3개 컴포넌트 전용 스타일, `#map`/`.timeline` 오프셋 보정 |
| `js/ui.js` | 모달 열기/닫기, 「오늘 하루 열지 않기」(localStorage+자정만료) / 「닫기」(sessionStorage) 분리 로직, 하단 푸터 버튼 핸들러 |

기존 파일 중 **지도 화면의 마커·레이어·검색·범례, 카드 UI, 이미지 시스템은 한 줄도
수정하지 않았다.** `.timeline`/`#map`의 위치만 푸터 높이만큼 보정했을 뿐, 동작 로직은
무변경이다(Playwright로 `navigateToEvent`/`slider`/`eraCard` 정상 동작 재확인).

## 알아두면 좋은 점

- **클라이언트 검증은 보조일 뿐**: `js/guestbook.js`의 글자수/URL 체크는 사용자 경험을
  위한 1차 안내고, 실제 차단(동일 IP 제한 등)은 전부 Worker(`guestbook-worker.js`)가
  수행한다. 브라우저 JS만으로는 우회가 trivial하기 때문이다.
- **금칙어 목록은 비워둔 채로 시작**: `BANNED_WORDS` 배열이 비어 있으면 그 검사는
  자동으로 비활성화된다. 운영하면서 필요한 단어를 그 배열에 채우면 된다.
- **featured(추천) 기록**: 관리자가 "추천 승인⭐"을 누르면 `featured:true`로 저장되고,
  공개 목록에서 항상 앞쪽에 정렬된다(작업지시서 6항 "최대 3개" 제한은 클라이언트
  렌더링 단계에서 추가로 자를 수 있다 — 필요하면 `guestbook.js`의 `renderItems`에
  featured 개수 제한 로직을 한 줄 추가하면 된다. 현재는 운영 중 실제로 몇 개나
  추천하게 될지 보고 정하는 게 나아 보여 강제 제한은 걸지 않았다).
