# 루트(Route) 웨이포인트 작성 규칙

루트는 ATLAS에서 지도 정주행이 아니라 "관심사 하나를 따라가는" 진입점이다
— 자료실과 함께 대부분의 방문자가 처음 만나는 콘텐츠가 될 가능성이 높다.
그래서 웨이포인트 하나하나가 그 자체로 "읽을거리"가 되어야 한다.

## 웨이포인트 스키마

```js
{
  id: 'wp_01',
  type: 'tragedy',           // WP_TYPE_LABEL/COLOR에 등록된 타입
  year: 1895, month: 10, day: 8,
  title_ko: '을미사변 — 명성황후 시해',
  place_ko: '한성 경복궁 건청궁',
  lat: 37.582, lng: 126.977,
  stay: null,                 // 선택: 김구 루트처럼 "체류 기간"이 있는 경우만
  summary_ko: '...',          // 선택: 2~4문장 정도. 아래 "언제 필요한가" 참고
  youtube_id: null,           // 선택: 유튜브 영상 ID 11자리만 (URL 전체 아님)
  card_ref: 'plot_1895_01',   // 필수: 지도 카드와 연결(클릭 시 카드 팝업)
}
```

## `summary_ko` — 언제 필요한가

**기본은 "넣는다"다.** 제목·장소만 보고 다 아는 사람은 소수다 — "여기서 무슨 일이
있었는지" 두세 문장만 있어도 훨씬 친절해진다. 카드(`card_ref`)에 이미 상세
설명이 있으니 완전히 새로 쓸 필요는 없다. 카드 요약을 2~4문장으로 축약하거나,
이 루트만의 관점(예: "이 인물이 이 시점에 왜 여기 있었는가")을 살려서 쓰면 된다.

**생략해도 되는 경우**: 제목 자체가 설명이거나(예: 단순 이동 경로의 중간
경유지), 같은 사건이 바로 위 웨이포인트에서 이미 충분히 설명된 경우. 이때는
`summary_ko: ''`으로 비워두면 — 그리고 `youtube_id`도 없으면 — 사이드 패널에
펼치기 화살표 자체가 안 뜨고 기존처럼 제목 클릭 시 지도 이동만 한다. 즉
"설명 없는 웨이포인트"는 특별히 처리할 게 없다, 그냥 비워두면 자동으로
간결한 모드로 동작한다.

## `youtube_id` — 넣는 법

전체 URL이 아니라 영상 ID(11자리)만 넣는다.

```
https://www.youtube.com/watch?v=dQw4w9WgXcQ  →  youtube_id: 'dQw4w9WgXcQ'
https://youtu.be/dQw4w9WgXcQ                  →  youtube_id: 'dQw4w9WgXcQ'
```

《기억의 연대기》 같은 자체 제작 영상이 특정 웨이포인트를 다룬다면 여기 연결한다.
영상은 실제로 펼쳐볼 때만 iframe이 생성된다(웨이포인트 수십 개짜리 루트를 열자마자
영상을 전부 로드하면 무거워지기 때문) — 즉 `youtube_id`를 넣는다고 초기 로딩이
느려지지 않는다.

## UI 동작 (js/routeRenderer.js, css/style.css)

사이드 패널에서 웨이포인트 제목을 누르면:
1. 지도가 항상 그 위치로 이동하고 마커가 깜빡인다 (`focusRouteWaypoint`) — 기존 동작 그대로.
2. `summary_ko`나 `youtube_id`가 있으면(`.has-detail` 클래스) 그 아래로 설명·영상이
   아코디언으로 펼쳐진다(`handleRoutePanelClick`). 없으면 1번만 일어난다 — 화살표
   아이콘 자체가 안 보인다.

이 로직은 **근대(`js/routeRenderer.js`)와 근현대(`maps/modern2/js/routeRenderer.js`)
양쪽에 동일하게 적용돼 있다** — 두 파일은 완전히 동일한 코드이므로, 이 파일을
고칠 일이 있으면 항상 두 곳 다 반영할 것.

## 아직 없는 것 — 중세1·중세2·고대·선사시대·현대의 루트

지금 루트는 근대(`routes/*.js`, 7개)와 근현대(`maps/modern2/routes/*.js`, 8개)에만
있다. 다른 지도(고려·조선·고대·선사시대·현대)는 아직 루트 자체가 없는 상태다.
콘텐츠가 쌓이면 같은 스키마·같은 UI 규칙으로 확장하면 된다 — 이번에 만든
아코디언 로직은 지도별 데이터 구조(왕대/블록)와 무관하게 웨이포인트 배열만
있으면 그대로 재사용된다.
