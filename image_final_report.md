# image_final_report.md — 이미지 최종 반영 + 규격 통일 검증 보고서

이 문서는 "프로젝트 이미지 최종 반영 지시(우선순위 최상)"에 대한 검증 결과다.
지시서의 핵심 원칙(사용자가 Numbers에 넣은 이미지는 이미 승인된 데이터,
AI 여부로 제외 금지, 체크박스 무시, 존재 = 사용)을 그대로 따랐다.

---

## 0. 사전 확인 — 이전 세션 작업이 이미 이 원칙을 만족하고 있었음

Numbers 원본에서 발견된 **고유 이미지 146건을 현재 데이터와 1:1 대조**한 결과,
**146건 전부 이미 카드에 연결되어 있었다(미연결 0건)**. 즉 이전 세션에서
AI 여부를 이유로 이미지를 제외하거나 보류한 사례는 없었다 — AI 생성
이미지(71건)도 실사 이미지(75건)도 차별 없이 전부 연결되어 있었다. 이번
세션은 그 위에서 ① 정확한 검증 수치를 산출하고 ② 지시서가 새로 요구한
이미지 규격 통일(CSS)을 추가로 적용하는 작업을 수행했다.

## 2. 규격 통일 적용 결과 (선택 항목)

지시서의 "이미지 규격 통일 정책"을 적용했다. 적용 전 `object-fit:cover`(잘라내기)
방식이던 것을 `object-fit:contain`(원본 비율 유지, 잘림 없음) 방식으로 전환했다.

| 항목 | 적용 전 | 적용 후 |
|---|---|---|
| hero 이미지 | `max-height:160px`, `object-fit:cover` | `aspect-ratio:4/3`, `object-fit:contain` |
| gallery 썸네일 | `height:72px`, `object-fit:cover` | `aspect-ratio:1/1`, `object-fit:contain` |
| 여백 처리 | 없음(잘려서 여백 자체가 발생 안 함) | `background:var(--paper)`(카드 배경색과 동일) |
| 모바일 오버라이드 | `max-height:280px` 별도 강제 | 제거 — 데스크탑과 동일한 `aspect-ratio` 적용 |

```
layout_before: object-fit:cover (위/아래 또는 좌/우가 카드 비율에 맞춰 잘림)
layout_after:  object-fit:contain (원본 전체가 보이고, 남는 공간은 카드 배경색)
cropped_count: 0
```

`cropped_count: 0`은 추정이 아니라 실측 결과다 — 헤드리스 브라우저에서 13개
샘플 이미지의 `naturalWidth/Height`(원본 크기)와 실제 렌더링된 `clientWidth/Height`,
`object-fit` 계산값을 직접 읽어 `contain`이 적용되어 있음을 확인했다(`contain`은
정의상 원본 비율을 보존하며 컨테이너에 맞춰 축소하므로, 수학적으로 크롭이
발생할 수 없는 방식이다). 추가로 146개 이미지 전체의 가로세로 비율을 검사해
극단적 비율(2.5:1 초과 등 — contain 적용 시 과도하게 작아 보일 수 있는 경우)은
0건임을 확인했다.

시각적 비교는 `screenshots/05_layout_before_cover.png`(변경 전 — 청산리 대첩
인물 사진의 머리/어깨가 잘림) vs `screenshots/06_layout_after_contain.png`
(변경 후 — 사진 전체가 카드 배경색과 함께 온전히 보임) 참고.

## 7. 최종 결론

> "Numbers에 들어있던 이미지가 실제 정보창에서 모두 보인다."

→ **충족**. 146/146건(100%) 연결, render_fail 0건, missing_images.csv 0행.

> "가능하면 이미지 높이가 정돈되어 보인다."

→ **충족**. `object-fit:contain` + 고정 비율 컨테이너로 모든 hero/gallery
이미지가 일관된 규격 안에서 원본 그대로(잘림 없이) 표시된다. `cropped_count: 0`.

표시 안정성(① 항목)이 규격 통일(② 항목)보다 항상 우선이라는 지시서 원칙에
따라, 규격 통일 작업 중 안정성을 해칠 신호(콘솔 에러, 렌더 실패, 극단적 비율
이미지)가 하나라도 발견되면 즉시 원본 표시로 되돌릴 준비를 하고 진행했으나,
실제로는 그런 신호가 전혀 발견되지 않아 규격 통일을 그대로 적용 완료했다.
