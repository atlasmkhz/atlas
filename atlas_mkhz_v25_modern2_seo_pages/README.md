# 기억의 연대기 (Chronicle of Memory)

**버전: v2 — 1876년 개항기까지 시대 확장 완료, 세계 컨텍스트 레이어 추가.**
**범위: 1876~1945년 한국 근대사. 세계 흐름(World Layer)은 같은 기간에 한정해 "넓고 얕게" 함께 표시.**

이 README가 설명하는 것은 "구조가 모듈화되어 검토 가능한 상태"라는 것이며,
"모든 기능이 완성됐다"는 뜻이 아니다. 두 가지는 다르다. 무엇이 구조적으로
준비되어 있고 무엇이 실제로 동작하는지는 5번(완성도)과 6번(미구현 기능)을
구분해서 읽을 것.

1876~1945년, 개항기부터 한국 독립운동을 거쳐 해방까지를 다루는 인터랙티브 역사 지도.
Leaflet.js 기반 지도 위에 64개 연도 파일·343개 사건·인물·정책 데이터를 연도별로 펼쳐 보여준다.
같은 기간 세계 각지(유럽·동아시아·중동·남아시아 등)에서 벌어진 흐름도 `data/world_events.js`를
통해 개요 수준으로 함께 표시된다(한국사는 깊게, 세계는 넓고 얕게 — 자세한 설계 의도는
해당 파일 상단 주석 참고).

---

## 1. 프로젝트 목적

이 프로젝트는 단순한 연표가 아니라 **"그 시대의 공간을 걷는" 역사 지도**를 지향한다.

- 연도를 슬라이더로 이동하면 사건이 나타나고 사라지며, 정책처럼 오래 지속되는 사건은 배경으로 남는다.
- 사건들은 인과관계(`connections`)로 서로 연결되어 있어, 화면에 흐름을 보여주는 점선으로 시각화된다.
- 사건은 전투·의열투쟁·정치·정책·학살·인물·민중운동·이동·조직결성 9가지 유형으로 분류되고, 레이어를 켜고 끌 수 있다.
- 학계 논쟁이 있는 사건(예: 훈춘사건의 배후, 자유시참변의 희생자 수)은 단정하지 않고 "다수 연구는 ~로 보지만 이견이 있다" 식으로 층위화해 서술한다.

장기적으로는 이 프로젝트가 선사시대부터 현대까지를 아우르는 더 큰 역사 아틀라스의 한 부분(근대 1876–1945 구간)이 될 것을 염두에 두고 설계되었다. 그 비전은 `docs/roadmap.md`에 별도로 정리되어 있으며, 현재 코드는 그 확장을 방해하지 않는 구조(연도별 데이터 분리, ID 기반 연결)를 따른다.

## 2. 실행 방법

이 프로젝트는 빌드 도구나 서버가 필요 없는 순수 정적 웹페이지다.

```bash
# 방법 1: 그냥 index.html을 더블클릭해서 브라우저로 열기
#         (단, 일부 브라우저는 file:// 프로토콜에서 script src 로딩을 막을 수 있음)

# 방법 2 (권장): 로컬 서버로 띄우기
cd chronicle
python3 -m http.server 8000
# 브라우저에서 http://localhost:8000 접속
```

외부 의존성은 Leaflet(지도 라이브러리)와 Carto 타일 서버뿐이며, 둘 다 CDN에서 로드한다. 인터넷 연결이 필요하다.

## 4. 의존성

| 항목 | 용도 | 로드 방식 |
|---|---|---|
| Leaflet 1.9.4 | 지도 렌더링 엔진 | CDN (jsdelivr) |
| Carto dark_nolabels 타일 | 배경 지도 이미지 (현재 활성 모드: world. legacy 모드인 dark_all로의 복귀 코드도 유지됨) | CDN (cartocdn.com) |

빌드 도구(webpack, npm 등) 없음. 순수 ES6 전역 스크립트로 구성되어 있어 `<script>` 태그 순서가 곧 모듈 의존성이다 (자세한 순서는 `index.html` 주석 참고).

## 5. 현재 완성도

- **데이터**: 1876~1945년 64개 연도 전체, 총 343개 사건. 모든 연도에 최소 1개 이상의 사건이 있다.
- **검증 통과**: 중복 ID 0건, 깨진 connections 0건, 좌표 유효성 100%, 생명주기 필드 오류 0건, 누락 연도 0건, type 스키마 외 값 0건, month 범위(0~11) 오류 0건 (2026년 6월 정규화 작업 기준).
- **핵심 기능**: 연도 슬라이더, 사건 팝업(지속기간 배지·관련 인물·유튜브 영상 포함), 레이어 토글(사건/인물/정책), 연결선 시각화, 마커 겹침 자동 해소(declutter), 干支 자동 계산, 시대별 색조 전환, 동해/독도 표기 보정, 세계 흐름(World Layer) 개요 표시(`data/world_events.js`).
- **모바일 대응**: Leaflet 라이브러리 로딩 실패 가드, iOS Safari 지도 컨테이너 크기 보정, 터치 제스처 설정.

## 6. 미구현 기능

다음은 설계는 되어 있으나 아직 코드에 반영되지 않은 항목이다.

- **검색 기능** — 인물명으로 검색해 해당 인물의 생애 동선을 추적하는 기능. (`docs/roadmap.md` 참고)
- **사실/해석/논쟁 4단 분리 구조** — 현재는 `summary_ko` 한 필드 안에 사실과 해석이 섞여 서술된다. 일부 사건(훈춘사건 등)은 본문에서 "다수 연구는 ~" 식으로 수동으로 층위화했지만, 데이터 구조 자체에 `facts`/`mainstream`/`debates`/`creator_note` 같은 필드는 아직 없다.
- **흐름(flow) 레이어** — 밀산→자유시, 상하이→충칭 같은 집단 이동 경로를 선으로 애니메이션화하는 기능.
- **인물 독립 인덱스(PERSON_DB)** — 현재 `people` 필드는 사건마다 이름 문자열 배열일 뿐, 인물별로 독립된 타임라인을 조회할 수 있는 구조가 아니다.
- **다국어 UI 전환** — 데이터에는 `title_en`, `title_ja` 필드가 모두 있지만, UI에서 언어를 전환하는 토글은 없다.
- **시대 확장(고대~1875년 이전, 1946년 이후 현대)** — README 1번에서 언급한 더 큰 아틀라스 비전 중 1876~1945 구간을 제외한 나머지 시대.

## 7. 알려진 버그 / 한계

- **world 모드는 코드상 적용되어 있으나 브라우저 렌더링 실측은 미검증이다.** `js/map.js`의 `MAP_MODE.current`는 현재 `'world'`로 설정되어 실제로 그 분기를 타며(legacy용 `seaLabelMask`/동해/독도 가리개 블록은 실행되지 않음), 동해·독도 표기 책임은 `js/historicalLabels.js`(`HISTORICAL_LABELS_ENABLED = true`, `places/places_min.js`의 `east_sea`/`dokdo` 항목)로 이미 넘어가 있다. 즉 구조적 의존관계(라벨 전환 시 historicalLabels.js도 같이 켜야 하는 부분)는 충족된 상태다. 다만 `dark_nolabels` 타일이 실제 브라우저에서 정상 응답하는지는 이 작업 환경의 네트워크가 cartocdn.com 접근을 차단해 직접 확인하지 못했다 — URL 패턴 자체는 CARTO의 공개 표준 스타일(`dark_all`/`dark_nolabels` 등)과 일치하지만, "코드가 그 분기를 탄다"와 "여러 줌 레벨에서 실제로 깨끗하게 렌더링된다"는 다른 문제다. 실사용 전 직접 브라우저에서 줌 레벨 2~11 전체를 육안으로 한 번 확인해 두는 것을 권장한다. legacy 모드(가리개 방식)로 되돌리는 코드는 그대로 남아 있어, 문제가 있으면 `MAP_MODE.current`를 `'legacy'`로 바꾸는 것만으로 즉시 복귀 가능하다.
- **세계지도 wrap 문제는 회피가 아니라 Leaflet 정식 기능(`worldCopyJump:true`)으로 해결되어 있다.** 과거에는 `maxBounds`로 시야를 동아시아~태평양 권역에 제한해 "가로로 한 바퀴 돌리면 마커가 사라지는" 상황 자체가 발생하지 않게 만드는 회피 방식(legacy 권역 제한)을 썼다. 2026년 6월 재작업에서 `maxBounds` 제한을 제거하고 `noWrap:true`를 빼고 `worldCopyJump:true`로 바꿔, 세계 어디로 이동해도(동/서 양방향) 끊김 없이 타일이 반복되고 마커도 사라지지 않도록 정식으로 고쳤다(`js/map.js` 36~69행 주석 참고). 단, 이 동작도 브라우저 실측 검증은 위 world 모드 항목과 마찬가지로 이 작업 환경에서는 하지 못했다.
- **연결선이 많아지면 성능이 저하될 수 있다.** 현재 343개 사건에서는 문제없지만, 향후 다른 시대를 추가해 사건 수가 크게 늘면 `renderer.js`의 `drawConnections()`가 매 렌더링마다 모든 선을 다시 그리는 구조라 프레임 드랍이 발생할 수 있다.
- **declutter(마커 겹침 해소)는 화면 좌표 기반이라 줌 변경마다 재계산된다.** 사건이 더 늘어나면 이 계산이 무거워질 수 있다.
- **모바일 미리보기 환경에서 외부 CDN 접근이 제한될 수 있다.** 실제 웹 호스팅 환경에서는 정상 작동하는 것으로 확인되었으나, 일부 인앱 브라우저(예: 채팅 앱 내장 미리보기)에서는 Leaflet 로딩이 실패할 수 있다.
- **`getImportance()`의 줌 레벨 기준이 하드코딩되어 있다.** 사건 수가 늘어나면 줌 레벨별 표시 기준(`zoom<5`, `zoom<6`)을 재조정해야 할 수 있다.
