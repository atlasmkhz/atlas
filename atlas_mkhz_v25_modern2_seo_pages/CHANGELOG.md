# CHANGELOG — README/데이터 정합성 작업

이 파일은 "README 기간 표기 불일치(1920→1876 누락), 사건/연도 수 오기, type·month 정규화,
1920년 source 보강" 작업 세션의 변경 내역만 별도로 기록한다. 그 이전의 누적 변경 이력은
`docs/changelog.md`를 참고할 것 — 그 파일은 건드리지 않았다.

작업 원칙(요청에 따름):
1. 실제 코드/데이터를 기준으로 문서를 맞춘다. 문서에 맞춰 코드를 억지로 바꾸지 않는다.
2. 동작이 실측 확인되지 않은 기능은 "지원됨"이라 적지 않고 "활성화됨/미검증"으로 표기한다.
3. 사건·인물 삭제 금지, UI/디자인 변경 금지, 파일 구조 대규모 변경 금지.

## 1단계 — README 정합화

- **기간 표기 수정**: `1920~1945` → `1876~1945` (실제 `data/` 파일 범위와 일치).
- **사건 수 수정**: `197개` → `338개` (실제 64개 연도 파일 전체 재계산 값).
- **연도 수 수정**: `26년` → `70년` (1876~1945, 실제 계산값).
- **World Layer 반영**: `data/world_events.js`(452행), `data/world_context.js`(74행)가
  이미 존재하고 동작 중인데 README가 "세계 아틀라스는 미구현"이라 적던 부분을 삭제하고,
  3번(폴더 구조)·5번(완성도) 섹션에 실제 역할을 기술.
- **검증 통계 문구 갱신**: "1920~1945년 26년 전체, 총 197개 사건" → "1876~1945년 64개 연도
  전체, 총 338개 사건". `validateMissingYears` 등 신규 검증 항목명도 함께 반영.

## 2단계 — 코드/문서 정합성 점검 (MAP_MODE)

`js/map.js`를 점검한 결과 **코드와 주석이 서로 다른 상태를 말하고 있었음**을 확인:

- `MAP_MODE.current`는 실제로 `'world'`로 설정되어 있고, 이 값에 따라 타일 URL이 실제로
  분기되며, legacy 전용 동해/독도 가리개 블록은 `current==='legacy'`일 때만 실행되므로
  현재는 미실행 상태였다.
- 그런데 주석들은 "현재 활성 모드: legacy", "world는 미구현"이라 적혀 있어 코드 실태와
  반대로 기술되어 있었다.
- 추가로 `js/historicalLabels.js`의 `HISTORICAL_LABELS_ENABLED`가 이미 `true`이고,
  `places/places_min.js`에 `east_sea`/`dokdo` 항목이 실존함을 확인 — world 모드 전환의
  구조적 의존관계(코드 주석이 우려하던 "기술 부채")는 이미 충족된 상태였다.
- **cartocdn.com `dark_nolabels` 타일이 브라우저에서 실제로 정상 렌더링되는지는 이 작업
  환경의 네트워크 제약(cartocdn.com 미접근)으로 검증하지 못함.** URL 패턴 자체는 CARTO
  공개 표준 스타일(`light_all`/`dark_all`/`dark_nolabels` 등)과 일치함을 외부 문서로
  확인했으나, "코드가 그 분기를 탄다"와 "여러 줌 레벨에서 실제로 정상 렌더링된다"는
  다른 문제이므로 — 원칙 2에 따라 **"지원됨"이 아니라 "활성화됨(코드 분기 적용 중),
  브라우저 렌더링 실측 미검증"으로 표기**.
- `MAP_MODE.current` 값과 타일 URL 등 **코드 자체는 변경하지 않음** — 주석/`description`
  문자열만 실태에 맞게 수정.
- "세계지도 wrap 문제"도 점검 결과 이미 `worldCopyJump:true` 등 Leaflet 정식 기능으로
  해결되어 있었는데(2026-06 재작업 주석 참고), README는 여전히 옛 "회피" 상태로 기술하고
  있어 함께 정정.
- 위 내용을 README 4번(의존성)·7번(알려진 버그/한계) 섹션에 반영.

## 3단계 — 스키마 정규화

### type / ID prefix
- 23건의 "ID prefix ≠ type" 불일치를 전수 검토. 참조 무결성(다른 사건의 `connections`가
  해당 ID를 참조하는 경우가 대부분)을 보존하기 위해 **ID는 변경하지 않고, type 필드를
  사건의 실제 내용 기준으로 재검토**.
  - 15건(`battle_1920_06`~`battle_1945_01` 등): 전부 "조직 결성/창설/재편/파견" 사건이라
    `type:'organization'`이 스키마 규칙과 정확히 일치 — 유지.
  - 7건(`political_1882_01` 등): 개별 검토 결과 현재 type이 사건 내용과 일치 — 유지.
  - **1건만 실제 오류로 수정**: `policy_1942_01`(조선어학회 사건) — 검거·투옥·고문이지
    집단살해가 아니므로 `type:'massacre'` → `type:'policy'`로 변경, `tags`도 동기화.
  - 판단 근거 전체는 작업 중 별도 기록(`type_decisions.md`, 세션 임시 파일)에 정리함.
- `institution_1880_01`(통리기무아문 설치): 스키마 9종에 없는 `type:'institution'` →
  `type:'organization'`으로 통합(기구 신설 사건이므로). `tags`도 동기화.
- `international_1881_01`, `international_1885_02`: 스키마 외 타입이었던 `international`을
  **10번째 정식 타입으로 schema.md에 추가**(국제정세/외교 사건, 조선이 주체가 아니거나
  외세 간 직접 충돌·교섭이 핵심인 사건). 데이터는 변경 없음.

### month
- `month:12`로 입력된 6건(1884, 1920×3, 1921, 1926×2)을 실제 역사적 날짜(예: 갑신정변
  1884년 12월 4일, 나석주 의거 1926년 12월 28일 — 모두 웹 검색으로 재검증)와 대조해
  스키마의 0-base 규칙(12월=11)에 맞게 `month:11`로 정규화.

### validator.js / schema.md
- `schema.md`: type 9종→10종 갱신, "ID prefix와 type" 설명 섹션 추가, month 필드에
  0-base 강조 및 흔한 실수 경고 추가, `source_type` 필드 신규 정의(`fact`/`interpretation`/
  `disputed`), 검증 규칙 요약 표 갱신.
- `js/validator.js`:
  - `validateMissingYears()` 기본 범위를 `1900~1945` → `1876~1945`로 수정(실제 데이터
    범위와 불일치하던 버그).
  - `validateEventTypes()`, `validateMonthRange()` 신규 추가 — 기존 함수는 변경하지 않고
    순수 추가, `validateAll()`에 결과 연결.

## 4단계 — 출처 보강

- **1920년 source 보강**: 34건 중 31건이 source 누락 상태였던 것을 전부 보강.
  한국민족문화대백과사전·우리역사넷·위키백과 등에서 각 사건을 실제로 검색해 검증한
  뒤 출처 배열을 채움(추측 출처 기재 없음). 결과: 1920년 source 보유 3/34(9%) →
  34/34(100%).
- **source_type 도입 및 적용**: schema.md에 필드를 정의하고, summary_ko 안에서 이미
  "다수 연구는 ~로 보지만 이견이 있다" 식으로 서술되어 있던 사건 중, **사실관계
  자체**(피해 규모, 배후, 책임 소재)에 학계 이견이 있는 9건에 `source_type:'disputed'`를
  부여:
  - `battle_1920_01`(봉오동 전투 — 일본군 사상자 수)
  - `plot_1920_01`(훈춘 사건 — 배후)
  - `massacre_1921_01`(자유시 참변 — 사망자 수·책임 소재)
  - `massacre_1923_01`(관동대지진 조선인 학살 — 희생자 수)
  - `person_1928_02`(박용만 피살 — 가해자 신원/배후)
  - `person_1930_01`(김좌진 암살 — 배후)
  - `massacre_1931_01`(평양 화교 학살 — 원인 해석)
  - `plot_1938_01`(남목청 사건 — 배후 미확정)
  - `migration_1945_02`(우키시마호 침몰 — 원인에 대한 이설)
  - 검토했으나 단순 서술이거나 사실관계 자체는 분명해 제외한 5건:
    `organization_1909_01`, `person_1914_04`, `political_1921_05`, `person_1930_02`,
    `person_1932_01`.
