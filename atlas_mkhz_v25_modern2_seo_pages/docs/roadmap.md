# Roadmap — 장기 확장 비전

이 문서는 현재 구현된 1920~1945년 근대 지도를 넘어서는 장기 비전을 정리한다.
**현재 코드베이스는 이 비전을 구현하지 않았다** — 아래 내용은 향후 별도 작업으로 진행될 계획이며, 지금의 README/architecture/schema 문서가 설명하는 실제 동작과는 구분된다.

이 문서를 별도로 둔 이유: 장기 비전이 매력적이라고 해서 지금 진행 중인 구체적 작업(파일 분리, 데이터 검증 등)과 뒤섞으면 둘 다 제대로 끝내지 못한다. 비전은 기록해두고, 지금은 한 가지씩 순서대로 완성한다.

---

## 1. 최종 목표

선사시대부터 현대까지를 아우르는 세계사·한국사 통합 역사 아틀라스. 현재의 근현대(1920–1945) 지도는 그 일부다.

핵심 질문:
- "사람들은 언제부터 자신을 한 나라라고 인식하기 시작했는가?"
- "공간, 문화, 전쟁, 기록, 민족, 국가의 경계는 어떻게 형성되었는가?"
- "한국사와 세계사는 어떤 방식으로 서로 얽혀 있었는가?"

## 2. 시대 구분 (제안)

| 시대 | 범위 | 표현 방식 |
|---|---|---|
| 선사시대 | 인류 등장 ~ 초기 국가 형성 이전 | 문화권·유적권·이동경로 중심. 국경선 지양. |
| 초기 고대·부족연맹 | 고조선, 부여, 옥저, 동예, 읍루, 말갈 등 | 지역·부족·연맹 단위. 현대 국가 개념 직접 대입 금지. |
| 고대 국가 | 고구려~발해, 백제, 신라, 가야, 통일신라 | 연도 기반. 전쟁·외교·도성이동 중심. |
| 중세 | 고려, 조선 | 왕조별→왕별→연도별 드릴다운 구조. |
| 근대 | 개항기~1945 해방 | **현재 구현된 부분.** 연도별 타임라인. |
| 근현대 | 1945~1993 문민정부 직전 | 분단·전쟁·산업화·민주화 중심. |
| 현대 | 1993~현재 | 세계화·디지털·국제관계. 현재성/검색 강화. |

## 3. 기술적으로 미리 고려해둔 것 (현재 코드에 반영됨)

장기 비전을 완전히 구현하지는 않았지만, 나중에 확장할 때 다시 갈아엎지 않도록 지금 코드에 다음을 반영해두었다:

- **연도별 데이터 분리 패턴**: `data/1920.js` 같은 패턴은 향후 `data/goryeo.js`, `data/prehistory.js`로 그대로 확장 가능. `app.js`의 DATA 조립 로직(`Object.values(DATA).flat()`)은 키가 연도(숫자)든 시대명(문자열)이든 상관없이 동작한다.
- **ID 기반 connections**: 시대가 늘어나도 제목 충돌 걱정 없이 연결 가능.
- **type 시스템의 확장성**: 현재 9종 외에 시대별로 다른 type(예: 선사시대의 `migration_culture`, `artifact_site`)을 추가해도 COLORS/TYPE_LABEL 객체에 키를 추가하는 것만으로 호환된다.
- **동해/독도 보정 원칙**: 이 원칙은 시대와 무관하게 항상 유지되어야 한다 — 어떤 시대를 추가하든 `map.js`의 보정 로직은 그대로 적용된다.

## 4. 아직 손대지 않은 큰 과제

### 4.1 시간 표현의 이질성
선사~고대 초기는 "연도"가 아니라 "구간"(구석기/신석기/청동기) 단위다. 현재 슬라이더는 정수 연도 1개를 입력받는 구조라, 이 시대를 추가하려면 슬라이더 자체를 "연도 모드"와 "시대구간 모드" 두 가지로 분기해야 한다. 작은 패치가 아니라 `timeline.js`의 재설계가 필요하다.

### 4.2 검색 기능 (인물 동선 추적)
설계 방향:
```javascript
// person index (가제)
{
  name: '김구',
  aliases: ['백범'],
  timeline: [...],       // 연도별 상태
  locations: [...],      // 거쳐간 장소
  event_refs: [...],     // 관련 사건 id 배열
  route: [[lat,lng], ...], // 이동 경로 (지도에 선으로 그릴 좌표)
  summary: '...'
}
```
현재 `people` 필드는 사건마다 이름 문자열만 있을 뿐 이런 구조가 없다. 별도의 `data/people_index.js` 같은 파일과, 그것을 조회하는 `js/search.js` 모듈이 필요하다.

### 4.3 사실/해석/논쟁 4단 분리
한사군 위치, 살수 위치, 고조선 영역 같은 논쟁적 주제를 다루려면 현재 `summary_ko` 한 필드에 모든 서술이 섞여 있는 구조로는 부족하다. 제안 구조:
```javascript
facts: ["사료로 확인 가능한 부분"],
mainstream: ["주류 학설"],
debates: ["대안설·논쟁"],
creator_note: ["제작자 관점 — 사실 단정과 명확히 분리해 표기"]
```
UI에서도 `[사실] [주류 해석] [논쟁] [제작자 관점]` 같은 탭이나 배지로 분리 표시해야 한다. 진보사학·재야사학계 관점을 배제하지 않되, "이것이 사실이다"와 "이런 관점이 있다"를 혼동시키지 않는 것이 핵심 원칙이다.

### 4.4 세계지도 확장 시 기술 문제

**현재 상태 명확화: `map.js`는 세계지도가 아니다.** `maxBounds`로 동아시아~태평양
권역에 지도를 묶어두었는데, 이것은 "세계지도를 만들었는데 일부만 보여준다"가
아니라 "세계지도가 가진 wrap 버그(가로로 한 바퀴 돌리면 마커 소실)를 일으킬
수 있는 조건 자체를 차단했다"는 뜻이다. 즉 문제를 해결한 것이 아니라 그 문제가
발생하는 상황을 지도 밖으로 밀어낸 것이다. `js/map.js`의 `MAP_MODE` 객체에
`legacy`/`world` 두 모드를 선언해두었으며, 현재는 `legacy`로 고정되어 있다.
세계지도로 확장하려면 다음을 모두 다시 설계해야 한다:

- 인도·태평양 부근에서 지도가 끊기는 문제(현재는 `maxBounds`가 그 권역에
  들어가지 않으므로 드러나지 않지만, bounds를 넓히면 재발할 것으로 예상되며
  아직 실제 재현 테스트는 하지 않았다)를 다시 검토해야 한다.
- 가로 wrap을 다시 허용할지(`noWrap:false`), 아니면 단일 세계지도로 유지할지 결정 필요.
- 마커/연결선/라벨이 경도 ±180도 경계를 넘을 때 사라지지 않도록 좌표 정규화 로직이 필요할 수 있다.
- "회색 가림판" 같은 임시방편이 아니라, 무라벨 베이스맵(라벨 없는 타일) + 우리가 직접 그리는 라벨 오버레이 구조로 가는 것이 근본적으로 더 안정적이다. 동해 표기 보정에 쓴 방식(`map.js`의 seaLabelMask)을 세계 전체로 확장하는 셈이다. 단, 현재의 동해 가리개 자체도 전 줌 레벨에서 검증되지 않은 임시조치이므로(`js/map.js` 주석 참고), 이 방식을 그대로 확장하기보다는 `dark_nolabels` + 직접 라벨링으로 처음부터 다시 설계하는 것을 권장한다.

### 4.5 시간 항해 모드 (실험적 기능, 토글형)
지구를 한 바퀴 돌리면 연도가 바뀌는 인터랙션은 흥미롭지만 기본 동작으로 강제하지 않는다. 구현하더라도 토글로 켜고 끌 수 있는 별도 모드로 분리해야, 기존의 "슬라이더로 연도 이동"이라는 안정적인 기본 경험을 해치지 않는다.

## 5. 작업 순서 제안

1. (완료) 1920~1945 근대 지도 데이터 완성 + 파일 구조 분리.
2. 사실/해석/논쟁 분리 구조를 근대 지도의 논쟁적 사건(훈춘사건, 만보산 사건, 보천보 전투 전과 등)에 먼저 시범 적용해보고, 구조가 검증되면 데이터 전체에 확산.
3. 인물 검색 기능을 근대 지도 범위(1920~1945) 안에서 먼저 구현 — 새 시대를 추가하기 전에 검색 구조부터 검증.
4. 시대구간 슬라이더 모드를 별도 프로토타입으로 먼저 만들어보고, 검증되면 기존 연도 슬라이더와 통합.
5. 중세(고려·조선)부터 시대를 하나씩 추가 — 근대와 가장 가까운 시대부터 거슬러 올라가는 순서가, 선사부터 내려오는 순서보다 검증하기 쉽다.
6. 마지막으로 세계지도 확장과 선사시대.
