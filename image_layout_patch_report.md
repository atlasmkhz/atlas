# image_layout_patch_report.md — 규격 통일 보완 (max-height 추가)

## 변경 내용

`.pop-image img`(hero)에 `max-height:420px`를 추가했다. `gallery`는 지시서가
명시한 그대로 변경하지 않았다(1:1 정사각형 썸네일은 카드 폭의 1/3 수준이라
420px에 도달할 일이 거의 없어 제한이 불필요).

```css
.pop-image img {
  width:100%; aspect-ratio:4/3; max-height:420px;
  object-fit:contain; object-position:center;
  background:var(--paper); display:block; border-radius:6px 6px 0 0;
}
```

## 동작 검증

| 시나리오 | 카드 폭 | 4:3 비율 시 기대 높이 | 실제 렌더링 결과 | 잘림 |
|---|---|---|---|---|
| 일반 패널(기존 환경) | 320px | 240px (420 미달) | 320×240 — max-height 영향 없음, 평소와 동일 | 없음 |
| 모바일 풀스크린 등 매우 넓은 카드 | 700px | 525px (420 초과) | 700×420 — max-height가 524px를 420px로 제한 | 없음(`object-fit:contain`이 그 안에서 원본 비율 유지) |

두 시나리오 모두 헤드리스 브라우저로 실제 렌더링 크기를 측정해 확인했다.
좁은 카드에서는 `max-height`가 전혀 작동하지 않아(4:3 비율이 이미 420px보다
작으므로) 기존 화면과 완전히 동일하게 보이고, 넓은 카드에서만 높이가 420px로
제한되면서도 이미지 전체가 잘리지 않고 카드 배경색 여백과 함께 표시된다.
스크린샷: 700px 폭에서 청산리 대첩 hero 이미지가 700×420으로 정확히 제한되고
사진 전체가 보임 — 별도 첨부 없이 본문 검증으로 충분하다고 판단해 스크린샷
파일은 이번엔 별도 생성하지 않았다(요청 시 추가 가능).

## 레이아웃 안정성 점검

지시서의 "contain 유지가 레이아웃 안정성을 해치면 되돌린다"는 조건에 따라
점검한 결과, 안정성을 해치는 신호(콘솔 에러, 잘림, 비정상적 카드 늘어짐)는
발견되지 않아 변경을 그대로 유지했다.

```
CSS 구문: 중괄호 208:208 일치
JS 구문 오류: 0건 (전체 77개 파일)
변경된 파일: css/style.css 1개뿐 (data/js/assets 전부 무변경)
```

## 결론

- 카드 높이가 지나치게 길어지는 경우(좁은 환경에선 발생하지 않음)에만
  `max-height:420px`가 작동해 스크롤 증가를 최소화한다.
- 어떤 경우에도 이미지는 잘리지 않는다(`object-fit:contain` 유지).
- 원본 비율은 항상 보존된다.
- 기존 사용 환경(좁은 패널)에는 시각적으로 아무 변화가 없다.
